# AutoBridge Mac

A native macOS app that bridges **ENET / DoIP (ISO 13400-2)** diagnostics over a TCP connection, with a local WebSocket tunnel so external tools can plug in.

Built with [Tauri 2](https://tauri.app/) · React 18 · Rust / Tokio.

---

## Features

- Select the active Ethernet interface
- Configure remote DoIP server IP, port, and DoIP logical addresses
- Optional session code for logging identification
- Live log panel with colour-coded DoIP message tracing
- Local WebSocket tunnel on `ws://127.0.0.1:8765` — connect external diagnostic tools
- macOS native window with traffic-light overlay

## Setup

```bash
# 1 – Clone / enter the project
cd autobridge-mac

# 2 – Install JavaScript dependencies
npm install

# 3 – Generate app icons (requires a 1024×1024 PNG source image)
npx tauri icon path/to/your-icon-1024.png
#   This writes all required sizes into src-tauri/icons/
#   Without real icons the build still works with Tauri's placeholder icons.
```

## Building the .dmg

```bash
npm run tauri build
```

The signed/unsigned artifacts appear at:

```
src-tauri/target/release/bundle/
├── dmg/   AutoBridge Mac_0.1.0_aarch64.dmg   ← drag-to-install disk image
└── macos/ AutoBridge Mac.app                  ← raw .app bundle
```

### Unsigned DMG (local testing)

Tauri builds without code-signing by default.  To open the unsigned DMG on
your own machine you may need to bypass Gatekeeper once:

```bash
xattr -cr "/Applications/AutoBridge Mac.app"
```

### Signed DMG (distribution)

Set the following environment variables before `tauri build`:

```bash
export APPLE_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
export APPLE_ID="you@example.com"
export APPLE_PASSWORD="app-specific-password"
export APPLE_TEAM_ID="YOURTEAMID"
```

Or configure them under `tauri.conf.json → bundle → macOS`.

## Using the WebSocket Tunnel

Once connected to a DoIP server, AutoBridge Mac listens on:

```
ws://127.0.0.1:8765   (configurable in the UI)
```

Send **binary** WebSocket frames — each frame is wrapped in a DoIP Diagnostic
Message and forwarded to the remote ECU.

Example with Python:

```python
import asyncio, websockets

async def send():
    async with websockets.connect("ws://127.0.0.1:8765") as ws:
        await ws.send(bytes([0x22, 0xF1, 0x90]))   # UDS ReadDataByIdentifier
        # DoIP ACK and response will appear in the app log

asyncio.run(send())
```

## Architecture

```
Frontend (React)
  │  invoke()  │  listen()
  ▼            ▼
Tauri commands  ←  Tauri events (log_entry, connection_status)
  │
  ├─ connect()  ──► enet_bridge::run_bridge()  ──TCP──► DoIP server
  │                       │
  │               tunnel::run_ws_server()  ◄── ws://127.0.0.1:8765
  │
  └─ disconnect()  ──► BridgeCommand::Disconnect
```
